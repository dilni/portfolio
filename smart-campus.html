University Project IIT & University of Westminster

Smart Campus – Sensor & Room Management

A scalable university infrastructure management platform designed to monitor campus facilities through intelligent sensor integration and centralized RESTful room management.

Distributed Systems & REST API Development

Project Overview

The system provides a RESTful API architecture that enables administrators to manage rooms, monitor environmental conditions (Temperature, CO₂), and process real-time sensor readings across multiple campus locations.

1. Core Backend API Component (The System Engine)

The backend serves as the central processing layer responsible for managing rooms, sensors, and environmental monitoring data.

RESTful API Architecture

Developed using Java JAX-RS (Jersey Framework). Implements resource-based endpoints for rooms, sensors, and readings with HATEOAS navigation.

Room & Sensor Management

Manages campus room records and tracks environmental monitoring sensors (Temperature, CO₂). Maintains operational statuses: Active, Maintenance, Offline.

Sensor Reading Processing

Records real-time environmental measurements dynamically. Stores timestamped information and supports sub-resource routing for sensor-specific readings.

2. Data Validation & Reliability Layer

Comprehensive validation mechanisms to ensure system consistency and reliability.

Exception Handling Framework

Centralized exception mapping architecture that generates standardized API error responses and improves debugging efficiency.

Resource Dependency Validation

Prevents sensors from being assigned to non-existent rooms and restricts deletion of rooms containing active sensors.

Thread-Safe Processing

Designed to support concurrent user access. Protects shared in-memory collections from race conditions, ensuring consistent behavior.

3. Hypermedia Navigation (HATEOAS)

Implements Hypermedia as the Engine of Application State (HATEOAS) to improve client-side navigation and discovery.

Dynamic Resource Discovery

API responses include navigational links, allowing clients to discover available resources dynamically without external documentation.

Self-Descriptive Responses

Contextual resource information improves API usability and scalability for integration with external applications.

4. Testing & Integration Component

The system was extensively tested using enterprise-grade API development and testing tools.

API Validation

Endpoint testing through Postman, including request-response verification and error scenario simulation.

Integration Testing

Validation of room-sensor relationships, reading submissions, and state transition for sensor statuses.
